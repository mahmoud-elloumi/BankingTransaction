1. VUE D'ENSEMBLE
================================================================================

Titre du Projet:
    TransactFlow - Real-Time Banking Transaction Monitoring System

Objectif:
    Créer un système de surveillance en temps réel des transactions bancaires
    utilisant Apache Kafka comme middleware de messaging et un dashboard web
    interactif pour visualiser et gérer les données de transactions.

Description:
    Le projet implémente une architecture événementielle complète permettant:
    - L'authentification sécurisée des utilisateurs
    - L'envoi et le suivi des transactions en temps réel
    - La détection automatique des fraudes
    - Les notifications par email
    - L'audit et le logging des opérations
    - Un dashboard web moderne et responsif

Status: ✓ Fonctionnel et prêt pour la présentation

10. FLUX DE DONNÉES KAFKA
================================================================================

PIPELINE DE TRANSACTION:

User Action (Envoyer Transaction)
              ↓
        DashboardServer (HTTP POST /api/send)
              ↓
    ┌─────────┴──────────┐
    ↓                    ↓
Kafka Producer      Kafka Topics
    │               ├── bank.transactions.pending
    │               ├── bank.transactions.approved
    │               ├── bank.transactions.rejected
    │               └── bank.transactions.fraud
    ↓
Kafka Consumers:
├── NotificationConsumer → Email SMTP (Gmail)
├── AuditLogConsumer → Log files & Audit trail
├── FraudDetectionConsumer → Risk scoring
└── DashboardConsumer → WebSocket/Polling Update
    ↓
  Frontend Dashboard (Real-time Feed Update)

TOPICS KAFKA CRÉÉS:

1. bank.transactions.pending
   - Transactions en attente de traitement
   - Partitions: 1
   - Replication Factor: 1

2. bank.transactions.approved
   - Transactions approuvées
   - Partitions: 1
   - Replication Factor: 1

3. bank.transactions.rejected
   - Transactions rejetées
   - Partitions: 1
   - Replication Factor: 1

4. bank.transactions.fraud
   - Transactions détectées comme frauduleuses
   - Partitions: 1
   - Replication Factor: 1

CONSOMMATEURS KAFKA IMPLÉMENTÉS:

1. NotificationConsumer
   - Envoie des emails via Gmail SMTP
   - Authentification sécurisée
   - Templates d'email personnalisés
   - Gestion des erreurs d'envoi

2. AuditLogConsumer
   - Enregistre tous les événements
   - Logs structurés en JSON
   - Timestamped pour traçabilité
   - Sauvegarde en fichiers

3. FraudDetectionConsumer
   - Analyse en temps réel
   - Scoring de risque
   - Détection montants anormaux (>50,000)
   - Alertes automatiques

4. DashboardConsumer
   - Met à jour les statistiques
   - Rafraîchit le feed en temps réel
   - Agrège les données
   - Communique avec le frontend

17. SUPPORT & TROUBLESHOOTING
================================================================================

PROBLÈME: Kafka ne démarre pas
SOLUTION:
- Vérifier que C:\tmp\kraft-combined-logs existe
- Vérifier les permissions d'accès
- Vérifier que le port 9092 n'est pas utilisé
- Nettoyer les logs: rmdir /s C:\tmp\kraft-combined-logs

PROBLÈME: Port 8080 déjà utilisé
SOLUTION:
netstat -ano | findstr :8080
taskkill /PID <PID> /F

PROBLÈME: Pas de transactions en temps réel
SOLUTION:
- Rafraîchir la page (F5)
- Vérifier la console Kafka
- Vérifier les logs du serveur
- Vérifier que Kafka est bien démarré

PROBLÈME: Erreur de connexion (login)
SOLUTION:
- Vérifier les credentials (voir tableau section 6)
- Vérifier que le serveur est bien lancé
- Vérifier la console pour les erreurs
- Essayer un autre compte

PROBLÈME: Emails ne sont pas envoyés
SOLUTION:
- Vérifier la configuration Gmail SMTP dans le code
- Vérifier que le compte Gmail a les app passwords activé
- Vérifier les logs pour les erreurs d'authentification
- Vérifier la connexion internet

LOGS UTILES:

Console Kafka:
- Chercher "[ControllerServer id=1]"
- Chercher "Awaiting socket connections"
- Chercher les Topics créés

Console Application:
- Chercher "[main] INFO"
- Chercher les requêtes HTTP
- Chercher les erreurs d'authentification
